## 摘要
基于大语言模型（LLM）的建筑法规自动化代码合规（ACC）方法容易生成错误且存在幻觉的计算机可处理规则。本文提出了P4IR，一个两阶段框架：首先使用监督微调（SFT）向LLM注入领域知识，然后通过组相对策略优化（GRPO）以高级代码骨架形式提高生成的中间表示的准确性。相对于SFT基线，该框架在树编辑距离和词元级莱文斯坦距离上分别减少了高达23.8%和38.6%。对比分析表明，该方法在零样本设置下，在代码结构和语义方面均优于领先的大语言模型，具体表现为通过少样本提示评估的Claude Opus和Sonnet 4.5、GPT-5.2、Qwen-3-Max以及GLM-4.7。此外，GRPO阶段还产生了虽小但统计上显著的假阳性减少。通过将SFT与GRPO相结合以直接优化领域特定目标，该方法为实现更准确、更可靠的基于LLM的ACC系统提供了一条途径。

## Abstract
Large language model (LLM)-based approaches for automated code compliance (ACC) of building regulations are prone to generating incorrect and hallucinated computer-processable rules. This paper introduces P4IR, a two-stage framework that uses supervised fine-tuning (SFT) to instill domain knowledge in an LLM, followed by Group Relative Policy Optimization (GRPO) to improve the accuracy of the generated intermediate representations in the form of high-level code skeletons. The framework achieved reductions of up to 23.8% and 38.6% in tree edit distance and token-level Levenshtein distance respectively, relative to the SFT baselines. Comparative analysis demonstrates that this approach in a zero-shot setting outperforms leading LLMs in both code structure and semantics, specifically Claude Opus and Sonnet 4.5, GPT-5.2, Qwen-3-Max, and GLM-4.7, evaluated via few-shot prompting. Additionally, the GRPO stage produced a small yet statistically significant reduction in false positives. By combining SFT with GRPO to optimize directly for domain-specific objectives, this approach offers a path toward more accurate and reliable LLM-based ACC systems.

- **核心问题**：基于大语言模型（LLM）的建筑法规自动化代码合规（ACC）系统，在将自然语言法规条文转换为计算机可处理规则时，容易生成结构不准确、包含幻觉（hallucination）的中间表示（Intermediate Representation，IR），进而导致后续合规检查失败。
- **研究动机**：传统ACC依赖手工编码规则，效率低且难以适应法规变更；LLM虽有潜力但存在幻觉问题，且监督微调（SFT）本身无法保证输出结构的正确性。现有研究（如GPT-3.5、GPT-4等）在少样本设置下仍难以生成与真实代码骨架在结构与语义上匹配的IR。
- **整体含义**：需要一种能够直接优化输出结构（代码骨架）的方法，在注入领域知识后，通过强化学习进一步减少幻觉、提高结构准确率，从而构建更可靠的LLM-based ACC系统。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：提出两阶段框架 **P4IR**（Policy Optimization for Intermediate Representations）：
    1. **第一阶段（SFT）**：将预训练LLM（Mistral 7B Instruct v0.3）在真实法规-代码骨架数据集上进行监督微调，使其具备领域知识，生成初始代码骨架。
    2. **第二阶段（GRPO）**：利用组相对策略优化（Group Relative Policy Optimization，GRPO）对微调后的模型进行强化学习，直接优化代码骨架的结构准确性，减少幻觉。
- **关键技术细节**：
    - **代码骨架（Code Skeleton）**：作为中间表示，仅保留类与函数的定义、参数、层级结构，函数体替换为`pass`，让模型专注于结构而非底层实现。
    - **GRPO**：使用G个候选输出（G=16）计算奖励均值与标准差作为基线，避免训练价值网络，降低计算开销；引入KL散度正则项防止偏离参考策略。
    - **奖励函数设计**：
        - **准确率奖励**（`R_structure`）：基于杰卡德相似度（Jaccard Similarity），比较生成与真实代码骨架的层级签名集合（类、函数、嵌套关系）。
        - **语法惩罚**（`R_syntax`）：若输出非合法Python代码则扣-1分。
        - **长度惩罚**（`R_length`）：若输出过短（低于`L_min`）且与目标长度差异超过阈值τ，扣-3分，防止奖励黑客。
    - **省略显式推理链**：论文认为在领域迁移下推理链可能误导，故不要求模型输出中间推理步骤。
- **公式（文字说明）**：
    - SFT损失：标准交叉熵，最大化给定输入检索目标代码骨架的条件概率。
    - GRPO目标：最大化裁剪后的策略比值优势，并减去KL散度项（β_KL=0.04）。
    - 优势计算：组内奖励归一化（`(r_i - mean)/std`）。
