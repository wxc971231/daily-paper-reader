## 摘要
我们提出了OptiAgent，一个多智能体框架，给定运筹学问题的自然语言描述，能够输出可直接由求解器处理的数学公式以及可执行代码。我们的架构优先考虑数学建模步骤，其中专门的智能体提取决策变量和约束等结构，从而实现迭代式自我修正。我们引入了一种新颖的多循环验证架构，包含四种专门的反馈机制，每种机制针对不同的故障模式，如误解、结构缺陷、数学不一致、验证失败和代码错误。除了准确性，我们的模块化设计通过提高透明度来改进求解优化问题的过程，因为每个智能体都会暴露其推理和反馈，使得整个建模过程可审计。我们的框架在LP、MILP和非线性规划任务的4个基准测试中的3个上达到了最先进水平，同时在剩余的数据集上仍具有很强竞争力。

## Abstract
We propose OptiAgent, a multi-agent framework that, given a natural language description of an Operations Research problem, is able to output a solver-ready mathematical formulation as well as executable code. Our architecture prioritizes the mathematical modeling step, where dedicated agents extract structures, such as decision variables and constraints, enabling iterative self-correction. We introduce a novel multi-loop validation architecture with four specialized feedback mechanisms, each targeting a distinct failure mode such as misinterpretation, structural defects, mathematical inconsistencies, validation failures, and code errors. Alongside accuracy, our modular design improves the process of solving optimization problems by improving transparency, as each agent exposes its reasoning and feedback, making the full modeling process auditable. Our framework achieves state-of-the-art performance on 3 out of 4 benchmarks across LP, MILP, and Nonlinear Programming tasks, while remaining highly competitive on the remaining dataset.

## 1. 论文的核心问题与整体含义（研究动机和背景）

运筹学（Operations Research, OR）问题的求解通常需要人工将自然语言描述转化为精确的数学模型（变量、约束、目标），再由求解器执行。这一过程专业门槛高、耗时且易错。大型语言模型（LLM）虽有潜力自动完成这一转化，但现有方法存在若干不足：
- 端到端黑箱过程，缺乏建模过程的透明度和可审计性；
- 难以自我纠正错误（如误解问题、结构缺陷、数学不一致、代码错误）；
- 生成的代码可能输出正确最优值却基于错误模型，埋藏风险。

因此，本文旨在构建一个**可解释、可迭代精化、模型无关**的多智能体框架，自动将自然语言问题描述转化为求解器可用的数学公式和可执行代码，并让用户能审查每个决策步骤。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
将优化建模任务分解为多个专门子任务，由六个专用智能体（Agent）按顺序流水线执行，并嵌入四个针对性的反馈循环，实现迭代自动纠错。

### 关键技术细节

**六个智能体**：
1. **解释智能体（Interpretation Agent）**：将自然语言问题转化为结构化表示（识别目标、实体、约束等），但暂不形式化。
2. **公式化智能体（Formulation Agent）**：将结构化表示转化为完整的数学模型，采用**五元素表示法**（集合、参数、决策变量、目标、约束）。
3. **分析智能体（Analysis Agent）**：根据变量类型、约束、目标判断问题类型（LP、MILP、NLP等），指导后续验证与求解器选择。
4. **验证智能体（Validation Agent）**：在语法（所有变量已定义）、语义（约束匹配意图）、数学（可行性和有界性）层面检查模型，返回APPROVED/REJECTED，并附具体反馈，指出应路由到哪个上游智能体。
5. **代码生成智能体（Code Generation Agent）**：将通过的数学公式转化为可执行Python代码（自动选择Pyomo、PuLP或OR-Tools库）。
6. **求解智能体（Solver Agent）**：在沙箱环境中运行代码，解析输出（状态、目标值、变量值），区分代码错误以路由回代码生成智能体。

**四个反馈循环**：
- **Loop 1**：验证智能体发现数学错误（如非线性假定LP）→ 路由回公式化智能体。
- **Loop 2**：公式化智能体发现根本性误解 → 路由回解释智能体。
- **Loop 3**：分析智能体发现结构问题（变量未定义、约束缺失）→ 路由回公式化智能体。
- **Loop 4**：求解智能体遇到语法或运行时错误 → 路由回代码生成智能体。

每个循环最多执行3次迭代，防止无限循环。

**共享状态**（AgentState）：累积中间输出、反馈信号、迭代计数器，流水线按依赖链执行。
