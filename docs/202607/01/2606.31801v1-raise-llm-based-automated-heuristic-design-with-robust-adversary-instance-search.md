## 摘要
自动启发式设计（AHD）与大语言模型（LLMs）的结合在发现高质量启发式算法方面取得了显著进展。然而，现有的基于LLM的AHD方法针对固定的训练实例集优化启发式算法，在现实世界分布变化部署时可能会灾难性地失败。我们提出了鲁棒对抗实例搜索（RAISE），这是一个将训练分布原则邻域内的约束最坏情况实例搜索集成到基于LLM的进化搜索循环中的框架。RAISE将鲁棒AHD视为一个约束对抗实例搜索问题：外循环通过LLM算子进化启发式算法，而无LLM的内循环使用带边界投影的基分布参数化，在训练实例集周围的epsilon球内高效识别困难实例。在在线装箱问题（OBP）、在线作业车间调度（OJSP）和在线车辆路径问题（OVRP）上跨五个分布族的全面实验表明，现有基于LLM的AHD方法在分布变化下性能下降达19倍，而RAISE在所有测试的分布和问题规模上持续保持强劲性能。

## Abstract
Automated Heuristic Design (AHD) with Large Language Models (LLMs) has shown remarkable progress in discovering high-quality heuristics. However, existing LLM-based AHD methods optimize heuristics for a fixed training instance set and may fail catastrophically when deployed under real-world distributional shifts. We propose Robust Adversary Instance Search (RAISE), a framework that integrates constrained worst-case instance search within a principled neighborhood of the training distribution into the LLM-based evolutionary search loop. RAISE treats robust AHD as a constrained adversarial instance search problem: the outer loop evolves heuristics via LLM operators, while an LLM-free inner loop efficiently identifies hard instances within an epsilon-ball around the training instance set using a basis distribution parameterization with boundary projection. Comprehensive experiments on Online Bin Packing (OBP), Online Job Shop Scheduling (OJSP), and Online Vehicle Routing (OVRP) across five distribution families demonstrate that existing LLM-based AHD methods degrade by up to 19 times under distribution shift, while RAISE consistently maintains strong performance across all tested distributions and problem scales

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **研究背景**：自动启发式设计（AHD）结合大语言模型（LLM）在发现高质量启发式算法方面取得重大进展，例如 EoH、ReEvo、PartEvo 等。这些方法利用 LLM 作为进化算子，迭代生成、重组和优化启发式函数。
- **核心问题**：现有 LLM 驱动的 AHD 方法仅针对**固定的训练实例分布**进行优化，生成的启发式高度特化。当部署环境中的问题实例来自未知或变化的分布（分布偏移）时，性能可能灾难性地下降——已有方法在测试中性能下降高达 **19 倍**。这一鲁棒性缺口严重限制了方法在真实世界动态环境中的可用性。
- **研究目标**：提出一种能够自动增强 LLM 驱动 AHD 鲁棒性，使其在训练分布附近的原则性邻域内有效应对最坏情况分布偏移的框架，同时无需依赖预定义的多样分布或多任务训练。

## 2. 论文提出的方法论
### 核心思想
- RAISE 将鲁棒 AHD 形式化为一个**约束极小极大优化问题**：

  \[
  h^* = \arg\max_{h \in \mathcal{H}} \min_{s' \in \mathcal{B}_\varepsilon(S)} \text{eval}(h, s')
  \]

  其中 \(\mathcal{B}_\varepsilon(S)\) 是以名义实例集 \(S\) 为中心、半径为 \(\varepsilon\) 的实例级不确定集（基于归一化 \(\ell_1\) 距离）。该形式化是分布鲁棒优化（DRO）的一个实例级可操作近似。

- 采用**双层进化搜索**框架：
  - **外循环（LLM 驱动的启发式进化）**：维护一个启发式种群，使用 LLM 算子（生成、交叉、变异）产生新候选；每个候选在当前的抗性实例集上通过鲁棒得分（实际使用均值聚合）评估；选择保留 top‑N 启发式。
  - **内循环（无 LLM 的对抗实例搜索）**：每隔 \(\tau\) 代（\(\tau=5\)）运行一次，针对当前最佳启发式，在 \(\varepsilon\) 球内寻找最坏实例。使用 **18 维基因向量**参数化候选分布，编码为**9 种基分布**（均匀、小、大、中心、双峰、高斯、周期、泊松类、峰值）的加权混合；通过**边界投影**确保实例落在 \(\varepsilon\) 球内。内循环采用均匀交叉和高斯变异进化 \(P_{in}=8\) 个候选，持续 \(G_{in}=4\) 代。最坏实例被追加到实例集 \(S\) 中，然后外循环重评分整个种群。

### 关键技术细节
- **实例级不确定集**：\(\mathcal{B}_\varepsilon(S) = \{ s \mid \exists s_i \in S, d(s, s_i) \le \varepsilon \}\)，其中 \(d\) 为归一化 \(\ell_1\) 距离。\(\varepsilon\) 控制鲁棒性强度：\(\varepsilon \to 0\) 退化为标准 AHD，\(\varepsilon \to \infty\) 变为完全分布无关的最坏情况优化。
- **外循环 LLM 算子**：使用五个算子（CREATE、E1、E2、M1、M2），分别用于探索结构新颖性和局部精化。
- **内循环投影**：将解码后的分布 \(p_{\text{adv}}\) 向其最近名义实例 \(b\) 的 \(\varepsilon\) 边界投影：\(p_{\text{proj}} = b + \frac{\varepsilon}{\max(\varepsilon, d(p_{\text{adv}}, b))} \cdot (p_{\text{adv}} - b)\)。这确保候选实例恰好位于边界上（最坏情况可能发生的位置）。
- **鲁棒得分**：内循环引入新实例后，外循环种群所有候选在该扩充集上重评分（使用均值聚合），以稳定选择。

## 3. 实验设计
### 任务与数据集
- **三个在线组合优化问题**：
  1. **在线装箱（OBP）**：目标最小化浪费率（废物比率 = (所用箱数−下界)/下界）。
     - 训练：5 个 Weibull 实例（\(n=5000, C=200\)）。
     - 测试：5 种分布族（均匀、正态、对数正态、指数、三角），3 种规模（\(n\in\{1000,5000,10000\}\)），4 种容量（\(C\in\{100,200,300,400\}\)），共 \(5\times3\times4=60\) 个数据集。
  2. **在线作业车间调度（JSP）**：目标最小化标准化完工时间（makespan）。
     - 训练：5 个均匀分布实例（10 机器×20 作业）。
     - 测试：5 种分布，2 种机器数（10,20），2 种作业数（20,50），共 \(5\times2\times2=20\) 个数据集。
  3. **在线车辆路径（VRP）**：目标最小化标准化路线长度比。
     - 训练：5 个实例（10 车辆×50 客户）。
     - 测试：5 种分布，3 种车队规模（5,10,15），共 \(5\times3=15\) 个数据集。
  - **总计 95 个测试数据集**，每个包含 5 个独立实例。
- **分布族**：Uniform、Normal、Lognormal、Exponential、Triangular，覆盖轻尾、重尾、有偏等典型模式。

### 对比方法（Baselines）
- **经典方法**：OBP 用 BestFit、FirstFit；JSP 用 SPT、MinSlack、EDD；VRP 用 Nearest-Feasible Insertion、Slack-Preserving Insertion、Urgency-Weighted Insertion。

- **鲁棒性感知 LLM 方法**：EoH‑S（基于 128 个多样实例的集成）、MoH（元优化+多任务训练）。
- **RAISE**：测试了四个 \(\varepsilon\) 值（0.001, 0.002, 0.005, 0.010），主配置 \(\varepsilon=0.002\)。

- 所有 LLM 方法使用 **GPT‑5‑mini** 作为骨干，总 LLM 采样预算 1000 次，种群大小 10。
